# Shop — Система управления интернет-магазином

CLI-приложение для управления интернет-магазином с программой лояльности и системой складского учёта.

Проект выполнен в рамках лабораторной работы по ООП на Python.

---

## 🏗️ Архитектура

Проект построен по принципу **Layered Architecture** с чёткой изоляцией слоёв:
```
┌─────────────────────────────────────────┐
│  presentation/  (CLI-интерфейс)         │
├─────────────────────────────────────────┤
│  service/  (Бизнес-логика)              │
├─────────────────────────────────────────┤
│  repository/  (Абстракция данных)       │
├─────────────────────────────────────────┤
│  domain/  (Модели, валидация, правила)  │
└─────────────────────────────────────────┘
          ↓
    PostgreSQL
```

### Структура проекта
```
lab_oop/
├── domain/              # Бизнес-сущности
│   ├── model.py         # Customer, Product, Order, OrderItem, Warehouse
│   ├── exceptions.py    # Иерархия исключений (ShopError → ...)
│   ├── validators.py    # Класс Validators
│   ├── strategies.py    # DiscountStrategy (паттерн Strategy)
│   ├── containers.py    # EntityCollection[T] (generic-контейнер)
│   └── utils.py         # Форматирование ID (C-001, P-001)
├── repository/          # Абстракция данных
│   ├── base.py          # Repository[T] (ABC Generic)
│   ├── postgres.py      # PostgreSQL реализация
│   ├── models_orm.py    # SQLAlchemy ORM модели (7 таблиц)
│   └── migrations/      # Alembic миграции
├── service/             # Бизнес-логика
│   ├── customer_service.py
│   ├── product_service.py
│   └── order_service.py
├── presentation/        # CLI интерфейс
│   └── cli.py           # Две панели: Пользователь/Админ
├── tests/               # Unit-тесты (100% покрытие domain)
├── db.py                # SQLAlchemy engine + session
├── main.py              # Точка входа
├── DATABASE.md          # Схема БД с диаграммой всех 7 таблиц
├── requirements.txt
├── alembic.ini
└── .env.example
```

## ✨ Особенности

### 1. Читаемые ID вместо UUID

Вместо длинных UUID (`6f9ccd22-476d-4adb-9e3d-...`) используются короткие ID:

- **Клиенты:** `C-001`, `C-002`, `C-003`
- **Товары:** `P-001`, `P-002`, `P-003`
- **Заказы:** `O-001`, `O-002`, `O-003`

Реализовано через автоинкремент в PostgreSQL + утилиты форматирования в `domain/utils.py`.

### 2. Две панели: Пользователь и Администратор

**Панель Пользователя:**
- Просмотр товаров с ценами и остатками
- Создание заказа от своего имени
- Просмотр только своих заказов

**Панель Администратора:**
- Управление клиентами (CRUD, повышение лояльности)
- Управление товарами (CRUD, активация/деактивация)
- Управление складом (добавление товара)
- Просмотр всех заказов
- Изменение статусов заказов

### 3. База данных в 3НФ — 7 таблиц

Схема нормализована до третьей нормальной формы:

- **`customers`** — клиенты с раздельными полями `first_name` / `last_name`
- **`categories`** — справочник категорий (вынесен из `products` для соблюдения 3НФ)
- **`products`** — товары со ссылкой на `category_id`
- **`orders`** — заголовки заказов
- **`order_items`** — позиции заказов (товар, количество, цена на момент заказа)
- **`warehouse`** — текущие остатки товаров (персистентное хранение в БД)
- **`archived_orders`** — история выполненных/отменённых заказов

Подробная схема с диаграммой связей — в [DATABASE.md](DATABASE.md).

## Настройка PostgreSQL

### Через Docker (рекомендуется)
```bash
docker run -d \
  --name shop_db \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=shop_db \
  -p 5432:5432 \
  postgres:16
```

### Создай `.env` файл
```bash
cp .env.example .env
```

Содержимое `.env`:
```env
DB_URL=postgresql://postgres:postgres@127.0.0.1:5432/shop_db
```

### Примени миграции
```bash
alembic upgrade head
```

## 📚 Использование

### Пример работы (Панель Админа)

**1. Создать клиента:**
```
Выбор: 2 (Админ) → 1 (Управление клиентами) → 1 (Создать)
Имя: Иван
Фамилия: Иванов
Email: ivan@example.com
Уровень лояльности: bronze

→ Создан: C-001 | Иван Иванов <ivan@example.com>
```

**2. Добавить товар:**
```
2 (Управление товарами) → 1 (Создать)
Название: Ноутбук
Цена: 50000
Категория: Электроника

→ Создан: P-001 | Ноутбук | 50000₽
```

**3. Добавить товар на склад:**
```
3 (Управление складом) → 1 (Добавить на склад)
ID товара: P-001
Количество: 10

→ На складе: 10
```

**4. Создать заказ (от пользователя):**
```
Выбор: 1 (Пользователь) → 2 (Создать заказ)
ID товара: P-001
Количество: 2

→ Заказ O-001 создан! Итого: 85000₽ (скидка 15% для gold)
```

## 🎓 Требования ТЗ

### ООП-концепции

- ✅ **Инкапсуляция:** Приватные поля (`_price`, `_loyalty_level`) с property
- ✅ **Композиция:** OrderItem содержит Product
- ✅ **Наследование:** Все исключения от `ShopError`, стратегии от `DiscountStrategy`
- ✅ **Полиморфизм:** `NoDiscount` / `LoyaltyDiscount` через единый интерфейс
- ✅ **Абстракция:** `abc.ABC` в `Repository[T]`, `DiscountStrategy`
- ✅ **Generics:** `Repository[T]`, `EntityCollection[T]`, `InMemoryRepository[T]`

### Функциональные требования

- ✅ **CRUD клиентов и товаров**
- ✅ **Учёт остатка товара** через `DBWarehouse` (персистентно в PostgreSQL)
- ✅ **Создание заказа** из нескольких позиций (`OrderItem`)
- ✅ **Скидки/бонусная система** через паттерн Strategy
- ✅ **Статусы заказа** (`Enum OrderStatus`)
- ✅ **Поиск, фильтрация, сортировка** через `EntityCollection[T]`
- ✅ **История заказов клиента** (`OrderService.get_customer_orders`)
- ✅ **Архивирование заказов** в `archived_orders`

### Технические требования

- ✅ **PostgreSQL** через SQLAlchemy + Alembic миграции
- ✅ **7 таблиц** в 3НФ (см. [DATABASE.md](DATABASE.md))
- ✅ **Модульность:** 4 слоя (domain, repository, service, presentation)
- ✅ **Пользовательские исключения:** 6 классов (`ShopError` → ...)
- ✅ **Магические методы:** `__str__`, `__eq__`, `__lt__`, `__len__`, `__iter__`, `__getitem__`, `__repr__`
- ✅ **Собственный контейнер:** `EntityCollection[T]` с методами filter/sort/find
