## Таблицы и их назначение

### 1. `customers` — Клиенты

Хранит профили покупателей с программой лояльности.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент, используется для ID вида `C-001` |
| `first_name` | VARCHAR | Имя (разделено с фамилией для соответствия 3НФ) |
| `last_name` | VARCHAR | Фамилия |
| `email` | VARCHAR UNIQUE | Электронная почта |
| `loyalty_level` | ENUM | `bronze` / `silver` / `gold` — влияет на скидку |
| `created_at` | DATETIME | Дата регистрации |

**Зачем:** Основная таблица пользователей. Разделение `first_name`/`last_name` устраняет транзитивную зависимость и позволяет сортировать/искать по каждому полю отдельно.

---

### 2. `categories` — Категории товаров

Справочник категорий, вынесенный в отдельную таблицу.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент |
| `name` | VARCHAR UNIQUE | Название категории (например, "Электроника") |

**Зачем:** Соблюдение 3НФ — если бы категория хранилась строкой прямо в `products`, то изменение названия категории потребовало бы UPDATE всех строк. С отдельной таблицей — один UPDATE.

---

### 3. `products` — Товары

Каталог товаров магазина.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент, используется для ID вида `P-001` |
| `name` | VARCHAR | Название товара |
| `price` | FLOAT | Текущая цена |
| `category_id` | INTEGER FK → categories | Ссылка на категорию |
| `is_active` | BOOLEAN | Признак активности (мягкое удаление) |

**Зачем:** Основная таблица каталога. Ссылка на `categories` вместо строки — 3НФ.

---

### 4. `orders` — Заказы

Заголовки заказов покупателей.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент, используется для ID вида `O-001` |
| `customer_id` | INTEGER FK → customers | Кто сделал заказ |
| `status` | ENUM | `pending` / `confirmed` / `shipped` / `delivered` / `cancelled` |
| `discount` | FLOAT | Процент скидки на момент создания заказа |
| `created_at` | DATETIME | Дата создания |

**Зачем:** Агрегат заказа. Скидка фиксируется на момент создания, чтобы изменение уровня лояльности не меняло старые заказы.

---

### 5. `order_items` — Позиции заказа

Декомпозиция заказа на отдельные товары.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | Автоинкремент |
| `order_id` | INTEGER FK → orders | К какому заказу относится |
| `product_id` | INTEGER FK → products | Какой товар |
| `quantity` | INTEGER | Количество единиц |
| `unit_price` | FLOAT | Цена на момент заказа (фиксируется) |

**Зачем:** Нормализует отношение "один заказ — много товаров". `unit_price` фиксируется при создании, чтобы изменение цены товара не меняло старые заказы.

---

### 6. `warehouse` — Склад

Текущие остатки товаров на складе.

| Поле | Тип | Описание |
|---|---|---|
| `product_id` | INTEGER PK FK → products | Ссылка на товар (1:1) |
| `quantity` | INTEGER | Текущий остаток |

**Зачем:** Персистентное хранение остатков в БД вместо in-memory словаря. При перезапуске приложения данные не теряются. Связь 1:1 с `products` — у каждого товара ровно одна запись склада.

---

### 7. `archived_orders` — Архив заказов

История заказов, завершённых или отменённых.

| Поле | Тип | Описание |
|---|---|---|
| `id` | INTEGER PK | ID оригинального заказа |
| `customer_id` | INTEGER FK → customers | Кто сделал заказ |
| `status` | VARCHAR | Финальный статус |
| `discount` | FLOAT | Применённая скидка |
| `total` | FLOAT | Итоговая сумма |
| `created_at` | DATETIME | Дата создания заказа |
| `archived_at` | DATETIME | Дата архивирования |

**Зачем:** Хранение истории без нагрузки на основную таблицу `orders`. Позволяет анализировать прошлые заказы, не засоряя активные данные. При архивировании строка из `orders` удаляется.

---

## Нормализация

| Правило | Как выполнено |
|---|---|
| **1НФ** — атомарные значения | `first_name` и `last_name` разделены; `unit_price` фиксируется в `order_items` |
| **2НФ** — нет частичных зависимостей | Все неключевые атрибуты зависят от всего первичного ключа |
| **3НФ** — нет транзитивных зависимостей | `category` вынесена в отдельную таблицу `categories`; `warehouse` — отдельная таблица |
